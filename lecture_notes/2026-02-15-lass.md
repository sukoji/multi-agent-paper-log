## 한 줄 요약

Agent state를 formal state space로 모델링해 transition과 invariant를 정의.

## 배경 · 문제 정의

Ad-hoc agent loop는 state mutation이 불투명하다. LASS는 planning·memory·tool을 unified state machine으로 표현한다.

## 핵심 방법

- State vector: beliefs, goals, tool outputs, messages
- Transition function typed by action category
- Invariant checks on state consistency
- Simulation harness for property testing

## 실험 · 결과

- Bug detection in agent workflows via invariant violation
- Formal comparison of ReAct vs plan-execute variants
- Case studies on customer support flows

## 한계 · 비판적으로 본 점

Full formalization overhead. Real LLM nondeterminism fits poorly.

## TIL — 내가 가져간 점

failure-modes taxonomy를 state transition에 매핑하면 재현성 좋아짐.
