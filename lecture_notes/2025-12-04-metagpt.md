## 한 줄 요약

SOP를 코드로 박아 PM·architect·engineer agent가 소프트웨어를 산출하는 meta-programming framework.

## 배경 · 문제 정의

ChatGPT-style 단일 대화로는 대형 SW 프로젝트 구조가 무너진다. MetaGPT는 인간 조직의 Standard Operating Procedure를 agent workflow에 인코딩한다.

## 핵심 방법

- Product Manager → Architect → Project Manager → Engineer 파이프라인
- 각 role 출력을 structured artifact(PRD, diagram, code)로 고정
- publish-subscribe message pool로 agent 간 공유
- executable feedback loop로 코드 검증

## 실험 · 결과

- MBPP 등 coding benchmark에서 multi-agent SOP가 단일 agent 대비 우수
- 완성된 repo 구조·문서·테스트까지 생성
- 오픈소스 구현으로 재현 가능

## 한계 · 비판적으로 본 점

토큰·시간 비용이 크고 SOP 변경에 유연하지 않다. Eval harness에 넣으려면 artifact 단위 checkpoint가 필요.
