## 한 줄 요약

서로 다른 vendor agent가 task를 delegate하기 위한 interoperability protocol.

## 배경 · 문제 정의

MCP는 tool wiring이고 agent간 협업은 별도 문제다. A2A는 agent card, task lifecycle, message format을 정의한다.

## 핵심 방법

- Agent Card JSON for capability advertisement
- Task object with state machine (submitted, working, done)
- Artifact and message exchange over HTTP/SSE
- Support human-in-the-loop escalation

## 실험 · 결과

- Initial partners: Salesforce, SAP, etc.
- Complements MCP in Google messaging
- Open spec and sample agents

## 한계 · 비판적으로 본 점

Early spec—ecosystem fragmentation risk with MCP-only stacks.
