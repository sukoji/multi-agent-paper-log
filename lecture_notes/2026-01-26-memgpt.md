## 한 줄 요약

Virtual context management로 main context와 archival storage를 OS처럼 스왑.

## 배경 · 문제 정의

Fixed context window는 long conversation과 document QA를 막는다. MemGPT는 LLM이 memory tier를 직접 manage한다.

## 핵심 방법

- Main context (RAM) vs archival (disk) vs recall buffer
- Function calls: core_memory_replace, archival_memory_search
- Queue-based message management for chat persistence
- Eval on long doc QA and multi-session chat

## 실험 · 결과

- Long document QA에서 retrieval baseline 대비 competitive
- Multi-day conversation coherence 개선
- Letta (구 MemGPT) OSS platform

## 한계 · 비판적으로 본 점

Self-edit memory가 잘못되면 corruption 누적. Multi-agent shared memory 미지원.
