## 한 줄 요약

Web navigation에서 출발해 desktop general control로 확장한 agent architecture.

## 배경 · 문제 정의

Web-only agent는 OS native app에 못 간다. Agent-E는 hierarchical DOM + OS UI unified approach를 시도한다.

## 핵심 방법

- High-level task decomposer
- Web: DOM tree pruning for token efficiency
- Desktop: accessibility + vision fusion
- Experience memory from past tasks

## 실험 · 결과

- WebArena subset competitive scores
- Desktop app tasks added in v2
- DOM pruning reduces tokens 40%

## 한계 · 비판적으로 본 점

Two-domain jack of all trades. Memory stale on UI updates.
