## 한 줄 요약

Exploration·demonstration phase로 앱을 학습한 뒤 autonomous execution.

## 배경 · 문제 정의

매 앱마다 수동 스크립트는 비현실적이다. AppAgent는 human demo 없이 self-explore로 UI affordance를 배운다.

## 핵심 방법

- Exploration: tap elements, record effect
- Documentation: build app-specific knowledge base
- Deployment: plan from natural language user goal
- Multimodal LLM for screen understanding

## 실험 · 결과

- Social, mail, map apps에서 50+ tasks
- Exploration phase 투자가 downstream success에 critical
- Cross-task within same app transfer

## 한계 · 비판적으로 본 점

Exploration time long. Cross-app workflow weak.

## TIL — 내가 가져간 점

mobile-agent 대비 learning phase가 차별점—eval에 cold vs warm start 분리.
