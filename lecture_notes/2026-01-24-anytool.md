## 한 줄 요약

API 검색·계획·self-reflection을 계층 agent로 묶은 large-scale tool system.

## 배경 · 문제 정의

ToolLLM 이후에도 긴 API chain에서 planning failure가 많다. AnyTool은 hierarchical decomposition과 self-reflective correction을 추가한다.

## 핵심 방법

- High-level planner breaks task into API subgoals
- API retriever with semantic search over ToolBench-scale catalog
- Self-reflection on failed calls before retry
- Hierarchical agent roles for plan vs execute

## 실험 · 결과

- ToolBench subset에서 pass rate 향상
- Reflection ablation 시 multi-step 실패 증가
- Scales to 16k API setting

## 한계 · 비판적으로 본 점

Reflection loop 비용 큼. Planner가 잘못 분해하면 하위가 복구 못 함.
