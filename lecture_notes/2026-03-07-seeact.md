## 한 줄 요약

Screenshot만으로 web action—grounding module이 핵심.

## 배경 · 문제 정의

Accessibility tree는 layout 정보를 잃는다. SeeAct는 visual grounding으로 element와 action을 연결한다.

## 핵심 방법

- Observe full page screenshot
- Propose action in natural language + target description
- Grounding model maps description to coordinates
- Set-of-mark prompting variant

## 실험 · 결과

- Mind2Web cross-domain strong results
- GPT-4V + grounding > text-only tree
- Grounding error dominant failure mode

## 한계 · 비판적으로 본 점

Screenshot token cost. Dynamic content timing issues.
